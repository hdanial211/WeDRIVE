2. Authentication & Verification Workflow

Detailed path of customer signups, credential matching, and document uploads in the welcome wizards.

Supabase Auth SDK 
flowchart TD
    USER["New or returning user"] --> ENTRY["account/pages/login/login.html"]
    
    ENTRY --> EMAIL_SIGNUP["Click Sign Up"]
    ENTRY --> EMAIL_LOGIN["Enter Email & Password"]
    ENTRY --> GOOGLE_LOGIN["Click Sign in with Google"]

    EMAIL_SIGNUP --> SIGNUP_UI["account/pages/signup/signup.html"]
    SIGNUP_UI --> CALL_SIGNUP["WeDriveAPI.signupUser()"]
    CALL_SIGNUP --> SUPABASE_SIGNUP["supabase.auth.signUp()"]
    SUPABASE_SIGNUP --> CREATE_CUST_RECORD["Create 'customers' record (status='Pending')"]
    CREATE_CUST_RECORD --> AUTO_CONFIRM["Auto-confirm authentication email (dev mode)"]
    AUTO_CONFIRM --> REDIRECT_WELCOME["Redirect to welcome.html"]

    GOOGLE_LOGIN --> SUPABASE_OAUTH["supabase.auth.signInWithOAuth(provider='google')"]
    SUPABASE_OAUTH --> OAUTH_CALLBACK["Google Callback Redirect"]
    OAUTH_CALLBACK --> CREATE_OAUTH_CUST["Verify / Insert record in 'customers' table"]
    CREATE_OAUTH_CUST --> REDIRECT_WELCOME

    EMAIL_LOGIN --> CALL_LOGIN["WeDriveAPI.loginUser()"]
    CALL_LOGIN --> SUPABASE_LOGIN["supabase.auth.signInWithPassword()"]
    SUPABASE_LOGIN --> LOGIN_CHECK_ADMIN{"Email exists in 'admins' table?"}
    
    LOGIN_CHECK_ADMIN -->|Yes| REDIRECT_ADMIN["Redirect to admin dashboard"]
    LOGIN_CHECK_ADMIN -->|No| REDIRECT_WELCOME

    REDIRECT_WELCOME --> WELCOME_JS["welcome.js checks verification status"]
    WELCOME_JS --> STATUS_CHECK{"Status in 'customers' table?"}

    STATUS_CHECK -->|Active| GO_CUST_DASH["Redirect to customer/pages/dashboard/customer.html"]
    STATUS_CHECK -->|Pending / Incomplete| GO_COMPLETE["Redirect to account/pages/complete-profile/complete-profile.html"]

    GO_COMPLETE --> UPLOAD_VERIFICATION["User uploads IC, License, formats Phone with country code"]
    UPLOAD_VERIFICATION --> SAVE_PROFILE["WeDriveAPI.updateCustomerProfile()"]
    SAVE_PROFILE --> UPDATE_DB_PENDING["Update status='Pending' in 'customers' table"]
    UPDATE_DB_PENDING --> GO_PENDING_PAGE["Redirect to verification-pending.html"]

6. Current Feature Implementation Status

Detailed checklist mapping our progressive integration to full operational status.

Active Status Checklist
System Module Status Technical Implementation details
User Authentication Active Supabase Auth integration. Password hashed via bcrypt, supporting email/password & Google OAuth 2.0.
Compliance Redirect Gate Active Autoredirect in welcome splash. Prevents incomplete or pending profiles from accessing the customer dashboard.
Fleet Inventory showcase Active Loaded from PostgreSQL cars table. Fully responsive columns with custom labels.
360 Virtual Inspection Active 200-frame sequential exterior view loader and Three.js panoramic interior render engine.
Calendar Booking blocking Active Flatpickr queries bookings table in real-time to identify conflict dates and gray them out.
Booking Checkout Active Processes payments in demo mode. Safely inserts verified records into Supabase bookings table.
Admin statistics cards Active Calculated live via aggregate queries: total revenue, active rentals, fleet, and monthly customers.
Admin Fleet management Active Complete CRUD operations (Add, Edit details, edit image references, Delete) syncing instantly to PostgreSQL.
Admin Booking control Active Search and date filters. Detail panel uses premium Stripe-style overlay modal with status dropdown overrides.
Admin Customer manager Active Customer profiles view showing license documents and toggle switch to block/suspend user.
System settings config Active Parameters (company contacts, tax rate, cancel policies) saved to settings table via UPSERT queries.
AI Support Chatbot Active LLM-powered chat assistant. Fetches context and credentials securely, fallback to Grok-3 if Gemini is offline.

7. Project File Map & Responsibilities

Identifies system files and how data structures are distributed in directories.

File Map
  • index.html - Main landing page and Auth entry point.
  • account/pages/login/login.html - User login form.
  • account/pages/signup/signup.html - User registration form.
  • account/pages/welcome/welcome.html - Splash gateway validating compliance.
  • account/pages/complete-profile/complete-profile.html - Profile completion document uploads.
  • customer/pages/dashboard/customer.html - Customer main page.
  • customer/pages/car-details/car-details.html - Immersive vehicle viewer details.
  • customer/pages/car-details/booking/booking.html - Date selectors with Flatpickr.
  • customer/pages/car-details/booking/payment/payment.html - RLS secure checkout.
  • customer/pages/my-bookings/my-bookings.html - Historical customer receipts.
  • admin/pages/dashboard/admin.html - Admin central statistics.
  • admin/pages/car/cars.html - Fleet inventory CRUD controls.
  • admin/pages/booking/bookings.html - Structured receipt details modal.
  • admin/pages/customer/customers.html - User authorization verification.
  • shared/js/api.js - Unified database API functions mapping.
  • shared/js/supabase-config.js - Core Supabase SDK initializer.
  • shared/js/main.js - Theme toggles and localized translations loader.
