2. Data and API Flow

Semua page tidak fetch data terus. Ia panggil pusat API iaitu shared/js/api.js.

Current: USE_REAL_DB false 
flowchart TD
    HTML["HTML page"] --> PAGE_JS["Page JS"]
    PAGE_JS --> API["shared/js/api.js - WeDriveAPI"]
    API --> SWITCH{"AppConfig.USE_REAL_DB"}

    SWITCH -->|false sekarang| JSON["shared/dummy/data.json"]
    SWITCH -->|true nanti| BACKEND["Backend API"]

    JSON --> SECTIONS["stats, car, bookings, customers, reports, settings, marketing"]
    BACKEND --> ENDPOINTS["/cars, /bookings, /customers, /auth/login, /admin/dashboard"]

    SECTIONS --> RENDER["Render UI"]
    ENDPOINTS --> RENDER
Data Section Kegunaan
stats Nombor ringkas dashboard admin.
car Senarai kereta, status, harga, spesifikasi dan gambar.
bookings Senarai tempahan customer.
customers Senarai pelanggan.
reports Revenue bulanan, utilization kereta dan summary report.
settings Maklumat syarikat, cukai, deposit, lokasi pickup.
marketing Banner, promo code dan seasonal pricing.

3. Login and Auth Flow

Login sekarang masih simulasi. Auth guard pula ada, tetapi dimatikan untuk development.

localStorage: wedrive_session 
flowchart TD
    USER["User isi email dan password"] --> HANDLE["handleLogin(event)"]
    HANDLE --> API["WeDriveAPI.loginUser()"]
    API --> DUMMY{"USE_REAL_DB false?"}

    DUMMY -->|Ya| CHECK{"Email contains admin?"}
    CHECK -->|Ya| ADMIN["Return role admin"]
    CHECK -->|Tidak| CUSTOMER["Return role customer"]

    ADMIN --> SAVE_ADMIN["Save wedrive_session"]
    CUSTOMER --> SAVE_CUSTOMER["Save wedrive_session"]
    SAVE_ADMIN --> GO_ADMIN["Redirect admin dashboard"]
    SAVE_CUSTOMER --> GO_CUSTOMER["Redirect customer dashboard"]

    DUMMY -->|Tidak| REAL["POST /auth/login"]
    REAL --> REAL_SESSION["Save real session"]
          
flowchart TD
    PAGE["Protected admin page"] --> GUARD["auth-guard.js"]
    GUARD --> ACTIVE{"AUTH_GUARD_ACTIVE"}
    ACTIVE -->|false sekarang| SKIP["Skip guard - page boleh buka terus"]
    ACTIVE -->|true nanti| SESSION["Check wedrive_session"]
    SESSION --> EXISTS{"Session wujud?"}
    EXISTS -->|Tidak| LOGIN["Redirect index.html"]
    EXISTS -->|Ya| ROLE{"Role match data-role?"}
    ROLE -->|Ya| ALLOW["Allow access"]
    ROLE -->|Tidak| LOGIN

flowchart LR

CustomerBoleh browse dan tekan Book Now, tetapi masih alert demo.

chatbot.js and chatbot-admin.js 
flowchart TD
    PAGE["Guest or customer page"] --> PLACEHOLDER["chatbot-placeholder"]

    JS --> UI["Inject chatbot panel and FAB"]
    UI --> OPEN["User opens chat"]
    OPEN --> SETTINGS["WeDriveAPI.getChatbotSettings()"]

    LOCAL -->|Ya| ADMIN_SETTINGS["Use admin saved settings"]
    LOCAL -->|Tidak| DEFAULTS["Use default replies"]
    ADMIN_SETTINGS --> MSG["User sends message"]
    DEFAULTS --> MSG
    MSG --> KEYWORD["Keyword match"]
    KEYWORD --> REPLY["Bot replies"]
          
flowchart TD
    ADMIN_CHAT["admin chatbot page"] --> LOAD["Load wedrive_chatbot_settings"]
    LOAD --> FORM["API keys, system prompt, promo context, greeting"]

    TEST --> GEMINI["Try Gemini first"]
    GEMINI --> OK{"Gemini success?"}
    OK -->|Ya| SHOW["Show reply"]
    OK -->|Tidak| GROK["Fallback to Grok"]
    GROK --> GROK_OK{"Grok success?"}

    GROK_OK -->|Tidak| FAIL["Show failed message"]
